# Чек-лист запуска: GSC · Vercel · ссылки · фото

> Внутренний документ для founder (RU разрешён для admin-доков; публичный UI — только EN).
> Задача: то, что нельзя сделать из кода и что сейчас реально мешает попасть на первую страницу Google.
> Домен-канон везде в коде: **`https://www.otherbali.com`** (с `www`). Держимся его.

Порядок по важности: **1) Vercel-домен → 2) GSC → 3) ссылки → 4) фото.**
(Сначала домен, потому что GSC и канонические ссылки должны указывать на один и тот же рабочий адрес.)

---

## 1. Vercel — закрепить `www.otherbali.com` как основной домен

Цель: чтобы `otherbali.com` (без www) автоматически перекидывал на `www.otherbali.com` (301-редирект). Тогда весь код, sitemap и Google смотрят на один адрес.

1. Зайти на **vercel.com** → войти → открыть проект **privelegy-bali-club** (он же Other Bali).
2. Верхнее меню проекта → **Settings** → слева **Domains**.
3. Проверить, что в списке есть **оба**: `otherbali.com` и `www.otherbali.com`.
   - Если какого-то нет — кнопка **Add**/**Add Domain**, ввести адрес, **Add**.
4. Навести на **`www.otherbali.com`** → у него должно быть **Set as Primary** (или он уже помечен *Primary*). Нажать **Set as Primary**, если не помечен.
5. У `otherbali.com` (без www) выбрать поведение **Redirect to `www.otherbali.com`** (Vercel обычно предлагает это сам — вариант «Redirect to primary domain»).
6. Если Vercel показывает **Invalid Configuration** и просит DNS-записи — их нужно добавить у регистратора домена (там, где покупали otherbali.com):
   - для `www` → запись типа **CNAME** на `cname.vercel-dns.com`;
   - для apex `otherbali.com` → запись типа **A** на IP, который покажет Vercel (обычно `76.76.21.21`).
   - Точные значения Vercel пишет прямо на этой странице — копировать оттуда.
7. Готово, когда у обоих доменов зелёная галочка **Valid Configuration**, и `otherbali.com` показывает **Redirect → www.otherbali.com**.

**Проверка:** открыть в браузере `otherbali.com` — адрес должен сам смениться на `www.otherbali.com`.

## 2. Google Search Console — Domain property + sitemap + индексация

Это ускоряет попадание новых страниц в Google с недель до дней. Три шага: подтвердить домен → отдать sitemap → попросить проиндексировать.

### 2.1 Добавить Domain property
1. Открыть **search.google.com/search-console** (тот же Google-аккаунт, что удобен).
2. Слева вверху — выпадающий список свойств → **Add property** / **+ Добавить ресурс**.
3. Выбрать **левый** вариант — **Domain** (не «URL prefix»). Ввести `otherbali.com` (без https, без www). **Continue**.
4. Google покажет **TXT-запись** для DNS (строка вида `google-site-verification=...`).
   - Добавить её у регистратора домена как запись типа **TXT** (там же, где делали DNS для Vercel).
   - Вернуться в GSC → **Verify**. (Иногда DNS обновляется до часа — если не сработало сразу, подождать и нажать Verify снова.)
   - Domain property покрывает сразу и `www`, и без-www, и http/https — поэтому он лучше, чем URL-prefix.

### 2.2 Отправить sitemap
1. В GSC слева → **Sitemaps**.
2. В поле «Add a new sitemap» ввести: `sitemap.xml`  → **Submit**.
   - Полный адрес: `https://www.otherbali.com/sitemap.xml` (у нас он живой и содержит все 63 страницы, все на www).
3. Через день-два статус должен стать **Success**.

### 2.3 Попросить проиндексировать ключевые страницы (Request Indexing)
Делать по одной, для самых важных новых страниц:
1. Вверху GSC — строка **URL inspection** (Проверка URL). Вставить полный адрес, напр.:
   - `https://www.otherbali.com/things-to-do-in-bali`
   - `https://www.otherbali.com/is-bali-safe`
   - `https://www.otherbali.com/nusa-penida-day-trip`
   - `https://www.otherbali.com/canggu`  · `/ubud` · `/uluwatu` · `/seminyak` · `/sanur` · `/nusa-dua` · `/jimbaran`
   - `https://www.otherbali.com/best-warungs-in-bali`
2. Enter → дождаться проверки → кнопка **Request Indexing** / **Запросить индексирование**.
3. Повторить для 10–15 главных страниц (лимит ~10–15 в день — это нормально, продолжить назавтра).

**Что смотреть потом (раз в неделю):** GSC → **Performance** — рост Impressions (показов) и позиций по запросам вроде «things to do in bali», «is bali safe», «warung canggu», «nusa penida day trip».

## 3. Обратные ссылки (backlinks) — первая волна через венью

Google верит сайтам, на которые ссылаются другие. Самый честный источник для нас — заведения, которые мы бесплатно и по-хорошему описали.

1. Открыть готовые шаблоны: **`docs/venue-outreach.md`** (письма/сообщения уже написаны, EN + RU).
2. Кому писать в первую очередь — у кого есть свой сайт/Linktree (ссылка с них весит больше всего). Начать с тех, кого мы уже вынесли на страницы районов.
3. Канал: WhatsApp / Instagram DM / email заведения.
4. Суть просьбы: «мы включили вас в гид Other Bali (ссылка на их страницу у нас) — будем рады, если добавите ссылку на otherbali.com у себя на сайте / в Linktree / в разделе "featured in"».
5. Цель первой волны — 10–20 отправленных сообщений. Даже 3–5 ответных ссылок заметно двигают.

> Важный нюанс по гардрейлам: мы не платим за ссылки и не обещаем платного продвижения. Только «вот полезная бесплатная страница про вас — сошлитесь, если считаете нужным».

## 4. Фото венью — по правилу согласия (v2)

Сейчас у части detail-страниц нет фото → они остаются `noindex` (это по нашему же правилу: **фото публикуем только с зафиксированным согласием владельца**, никакой подмены стоковыми картинками).

Чтобы открыть эти страницы для индексации, на каждое заведение нужно:
1. Получить **фото** от владельца (или разрешение снять/использовать конкретные фото).
2. Зафиксировать **запись согласия**: кто дал · когда · версия контента · канал · подтверждение прав на фото. (Полный процесс — `docs/DATA_OPS_TRACK.md`.)
3. После этого фото можно публиковать, и страница становится индексируемой.

Это медленнее, но это самый большой оставшийся контентный рычаг после индексации.

## Быстрый порядок на сегодня (30–40 минут)

1. **Vercel** → Domains → `www.otherbali.com` = Primary, apex = Redirect. (5–10 мин + возможно DNS)
2. **GSC** → Add property → Domain → TXT в DNS → Verify. (10 мин + ожидание DNS)
3. **GSC** → Sitemaps → `sitemap.xml` → Submit. (1 мин)
4. **GSC** → URL inspection → Request Indexing на 10–15 главных страниц. (10 мин)
5. Позже на неделе: разослать первую волну писем венью из `docs/venue-outreach.md`.

> Если на шаге с DNS (Vercel TXT/CNAME/A или GSC TXT) будет непонятно — сделай скриншот страницы регистратора домена и пришли мне, распишу точные значения под твоего регистратора.
