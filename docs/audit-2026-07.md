## Added launch foundation in this worktree

- Five workflow definitions: quality CI, preview smoke, deterministic data audit, unsigned iOS Release, and production-build Chromium/axe Browser E2E. The browser workflow has a secret-free PR/push job plus a protected-ref, environment-scoped isolated-staging job; neither is claimed live for this unpushed diff.
- Runtime schemas, publication gates, additive migrations `0035`–`0041`, normalized venue/photo backfill review, service-only privacy RPCs, consent-backed event throttling and hardened same-contact guide-lead submission.
- Consent enforcement, Privacy Choices, export/delete RPCs and a factual data inventory.
- Bounded event/source APIs, active source registry, atomic GuestRef rate limit and unified external URL policy. GuestRef is minted only behind a Web Locks cross-tab critical section and signed with a server-side HMAC proof; ordinary documents and Essential-only choice remain identity-free. Browsers without a reliable Web Locks primitive fail closed for identity-bearing actions.
- Real 404/error states, request IDs, release-aware safe logs and live/ready endpoints.
- Commercially neutral Places ranking: fit precedes completeness, and tier/sponsored state does not affect organic order. Active criteria and bounded browser-history filters mirror the user's actual query, district and category choices.
- Exact fail-closed route integrity plus ordered route pages, stored Google Maps handoffs labelled as exact place, search or generic according to evidence, save/share, related-route navigation and bounded incorrect-info reporting. No synthetic stops, live routing, ETA or provider travel-time data is presented.
- Versioned Mobile API v1 plus a bundled local Capacitor shell using App, AppLauncher, Browser, Network, Preferences and Share for exact local URL routing, native network state, controlled external handoffs, native share and Preferences-backed state.
- Bounded venue detail and full ordered saved-route snapshots commit as combined atomic records and can reopen offline from Preferences. Unreadable authoritative or migration storage fails closed and cannot be overwritten by an assumed empty state. This does not imply offline fulfilment, live availability or background sync.
- Save state is idempotent, source capture is atomic, shared lists are owned, and deletion leaves only a hash-only tombstone barrier so a deleted GuestRef cannot be recreated by a late/in-flight write. Tombstone retention remains an owner/legal decision.
- Token-scoped onboarding writes are venue-serialized and bounded. Listing confirmation is first-write-wins; normalized exact owner-note retries are no-write; changed owner notes, menu drafts and action drafts have durable venue-wide daily/lifetime budgets, and menu/action drafts also have a fixed pending ceiling.
- Venue photo submission is a private, consent-gated state machine. Upload bytes are SHA-256-bound to the evidence row; only an uploaded exact digest with exact consent can enter review or approval. Ambiguous upload/consent responses are retained for reconciliation rather than guessed, direct application DML is revoked, and a bounded dry-run-by-default cleanup operator waits through a grace period before removing unconsented objects.

## Honest release verdict

- Website: launch foundation and product-integrity slices passed the pre-merge local credential-free gate and independent P0/P1 review, but the merged tree still requires live CI and a configured production-like preview; production is not proven by the merge itself.
- iOS: six native plugin integrations, local cold/warm URL handlers and offline public snapshots are implemented; signing, associated-domain/AASA delivery and native map evidence remain open, so it is not TestFlight/App-Store-ready.
