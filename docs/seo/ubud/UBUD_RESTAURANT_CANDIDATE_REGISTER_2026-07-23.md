## Ограничения и публикационные гейты

1. До создания любой `/places/...` страницы нужно подтвердить Google Maps entity, точный адрес, часы, меню и статус “open”.
2. Для hotel-ресторанов (Kubu, Swept Away и аналогов) нужен owner/hotel confirmation и проверка, принимают ли они внешних посетителей.
3. Для локальных warung без официального сайта обязательны field-check и фото/чек/контакт владельца.
4. Не использовать claims “лучший”, “№1”, “must-visit” без отдельного evidence-пакета.
5. После закрытия гейтов кандидаты должны быть распределены между `/ubud/best-restaurants` и самостоятельными place pages; новые URL создаются только отдельным разрешением.

Источники: официальные сайты, перечисленные в таблице; discovery-списки использовались только для формирования кандидатов и не считаются доказательством качества.
