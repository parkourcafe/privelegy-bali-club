## Implemented in this integration worktree

- Canonical origin is `https://www.otherbali.com`; apex redirects permanently while preserving path/query. Sitemap, robots, metadata, mobile config and absolute URLs use `www`.
- Five workflow definitions cover quality, preview HTTP smoke, deterministic data audit, unsigned iOS Simulator Release, and production-build Chromium/axe browser gates. Browser PR/push runs are credential-free; configured data-backed proof is a separate manual job restricted to a protected ref and the `otherbali-browser-staging` environment.
- Runtime venue/perk/route/mobile DTO schemas fail closed. Invalid public rows are excluded with structured issue codes; fixture fallback is limited to explicit local development.
- Additive migrations `0035`–`0041` normalize venues/photos, add service-only privacy export/delete/consent RPCs, add consent-backed atomic event rate limiting, harden guide-lead submission, make save/source writes idempotent/atomic, enforce shared-list ownership, add a hash-only GuestRef deletion tombstone barrier, bound token-scoped onboarding writes and install the reconciled private-photo lifecycle.
- Migration `0040` makes listing confirmation first-write-wins and serializes owner-note/menu/action proposals per venue. Normalized exact retries are no-write. Owner-note changes are capped at 10/24h and 50 lifetime; menu/action proposals are capped at 5 pending, 10/24h and 50 lifetime with private append-only/durable hash ledgers that survive rejected-draft deletion and token rotation.
- Migration `0041` stages private photo uploads before review, binds the object bytes by SHA-256, records exact consent under the same row lock, revokes direct application DML and fails readiness on unresolved/missing/stale/inconsistent lifecycle state. Ambiguous upload acknowledgement is request-only cleanup plus `202`; the operator removes it only after an immutable cleanup timestamp passes a configured grace period and a fresh consent-aware CAS succeeds.
- Privacy has `unknown`, `essential_only` and `analytics_allowed`; ordinary page requests create no GuestRef. GA is disabled by default for the first build and would still require explicit browser consent if enabled later.
- `/privacy/choices` supports consent withdrawal, analytics-cookie cleanup, export and confirmed deletion. Identity-bearing actions use Secure host-only `__Host-` GuestRef/HMAC cookies; duplicate protected names fail closed. The sole minting endpoint is ordered by Web Locks and fails closed where Web Locks are unavailable. Expiring storage leases are deliberately excluded because tab suspension can split ownership. Unprefixed legacy bearer cookies are neither trust-migrated nor silently replaced: identity actions return migration-required while preserving the handle for a separately approved support/migration/erasure procedure.
- Event/source bodies are bounded and schema-validated. Client-supplied identity/source fields are rejected, campaign sources use the active DB registry, and public errors omit database details.
- Places ranking is commercially neutral: fit is evaluated before completeness, and neither partner tier nor sponsored state contributes to organic order. The UI exposes only actual active criteria.
- Place search, district and category state is synchronized with bounded native browser history: typing replaces the current entry, deliberate filter changes push an entry, Back/Forward restores the filters, and Clear removes stale planner parameters.
- Route definitions are resolved exactly and fail closed as a whole when a stop is missing, unpublished, malformed or duplicated. No synthetic category or first-unused stop is substituted.
- `/route/[slug]` renders the ordered editorial sequence with linked stops, stored Google Maps handoffs labelled as exact place, search or generic according to URL evidence, route save and share controls, and honest absence of live hours, ETA, routing or provider travel times. Venue pages expose only exact related routes.
- Venue detail includes a bounded `Report incorrect info` mail handoff with six explicit reasons. No free-text report is stored or presented as already submitted by Other Bali.
- Mobile API v1 exposes strict versioned public DTOs for config, bootstrap, venues and routes, with ETag, safe CORS and fail-closed dependency handling.
- Capacitor Release uses bundled `ios-web` assets with no production `server.url` and no Release bridge logging. The exact Capacitor App, AppLauncher, Browser, Network, Preferences and Share plugins are declared and used for cold/warm local URL routing, native network state, Preferences-backed storage, controlled external handoffs and native sharing.
- The local shell persists each saved-ID/snapshot set as one bounded atomic Preferences record, including source timestamps, so saved routes and their compact stop details can reopen offline. Writes are serialized; unreadable native/legacy storage blocks mutation and offers retry instead of overwriting an unknown state. This is local public-data caching, not offline booking, background sync or live availability.
- Root and key public route error boundaries, a real 404 surface, request correlation IDs, safe structured failure logs and live/ready health endpoints are present.
- `/api/health/ready` fails closed unless public district data has a positive integer count, the current GuestRef signing secret is valid, the service credential works and the exact service-only `release_readiness_v2` payload is `{ok:true,version:2,schemaRevision:"0041"}`; it returns no tenant data or private dependency detail.
- Shared Basic Auth cannot expose `/admin` on the public production deployment.
