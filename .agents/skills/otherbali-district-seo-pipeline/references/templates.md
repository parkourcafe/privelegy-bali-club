# District document templates

## Inventory and intent map

Include date, scope, existing routes, registry/sitemap/canonical state, internal links, SERP query clusters, user job, competing URLs, result types and evidence labels.

## Unified cluster decision

Use columns: `topic`, `primary intent`, `existing owner`, `decision`, `target URL`, `evidence readiness`, `assumption`, `confidence`, `blocker`, `next action`.

## Content brief

Include page ID/URL, action, primary intent, audience, decision output, title/H1/meta, approved claim IDs, prohibited claims, outline, internal links, schema, canonical, sitemap/indexing and readiness.

## Owner confirmation pack

For each entity record recipient/channel, exact questions, affected claim/topic, expected evidence, draft status and send prohibition unless separately authorized.

## Field verification pack

For each location record exact entrance/pin, date/time/WITA, observer, method, signage/access/safety observations, media paths, affected claim, result and reviewer.

## Final report

Report: skill/pipeline; agents; existing pages; CREATE/UPDATE/MERGE/HOLD/REJECT; implemented P0 pages; used claims; blockers; owner/field/Maps checks; tests/build/QA; branch; commit; preview URL/state; diff summary; next district.
